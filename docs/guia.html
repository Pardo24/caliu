Guia d'usuari de Gecko

Aquesta guia explica com fer servir Gecko pas a pas: des del primer inici fins a accedir als teus serveis des de qualsevol lloc del món.

⬇ Descarregar Gecko ← Pàgina principal

🚀 Primer cop

Quan obres Gecko per primera vegada (o si mai has instal·lat el stack), l'aplicació mostra l'assistent d'instal·lació — un wizard pas a pas que configura tot automàticament. Si ja tens el stack instal·lat, Gecko detecta la configuració existent i obre el dashboard directament.

ℹ️
Requisit previ: Cal tenir Docker Desktop instal·lat i obert. És gratuït. Gecko comprova automàticament si Docker funciona i et redirigeix si cal instal·lar-lo.

🧙 L'assistent d'instal·lació

L'assistent guia l'usuari en 5 passos. Una barra de progrés a la part superior indica en quin punt ets.

1

Benvinguda

Pantalla introductòria amb el resum de què fa Gecko. Prem Començar per continuar.

2

Comprovació de Docker

Gecko comprova si Docker Desktop és actiu. Si no el tens, veuràs un botó per descarregar-lo. Un cop instal·lat Docker, prem Ja l'he instal·lat, torna a comprovar.

3

Carpeta de dades

Selecciona on es guardaran els fitxers de configuració i els continguts que descarreguis. Recomana un disc extern de mínim 1 TB. Gecko no esborra cap fitxer existent.

4

Contrasenya i idioma de subtítols

Crea la contrasenya per als serveis (usuari sempre admin, mínim 6 caràcters). Tria els idiomes de subtítols que vols — Gecko configura Bazarr automàticament amb la teva selecció.

5

VPN (recomanada)

Activa una VPN WireGuard (ProtonVPN o Mullvad) per protegir les teves descàrregues. Sense VPN l'ISP pot detectar el tràfic P2P. Pots configurar-la després des del dashboard si ho prefereixes.

✓

Instal·lació i auto-configuració completa

Gecko descarrega tots els serveis i els connecta automàticament entre ells — Prowlarr integrat amb Radarr, Sonarr i Lidarr; qBittorrent, Jellyseerr i Bazarr configurats. Pot trigar 5-10 minuts la primera vegada. L'únic que hauràs de fer manualment és afegir indexadors a Prowlarr.

✅
Configuració zero manual: A diferència d'altres solucions, Gecko connecta tots els serveis automàticament. Un cop acabat l'assistent, l'únic pas manual que queda és afegir indexadors a Prowlarr. Com fer-ho →

🗂️ El menú lateral del dashboard

Un cop instal·lat el stack, Gecko mostra sempre el dashboard amb una barra lateral de navegació a l'esquerra amb quatre seccions:

🏠
Inici
Estat del stack (en marxa / aturat), botons per arrencar i parar tots els serveis, i accés ràpid a cada servei amb un clic.
🔒
VPN
Veure si la VPN és activa. Activar ProtonVPN o Mullvad introduint les claus WireGuard. Desactivar-la si és necessari. Els canvis s'apliquen reiniciant el stack automàticament.
🌐
Xarxa
La teva IP local a la xarxa Wi-Fi, les URLs de cada servei per xarxa local, i informació per accedir des de fora de casa amb Tailscale.
⚙️
Configuració
Canviar l'idioma de l'aplicació (català, castellà, anglès). Veure la versió de Gecko. Reinstal·lar el stack des de zero (no esborra el contingut multimèdia).
⬇️
Descàrregues
Veu en temps real les descàrregues en curs de Radarr i Sonarr: progrés, mida i temps restant. Per cada ítem pots fer blocklist + nova cerca si la descàrrega no t'interessa.

Idioma: A la part inferior de la barra lateral hi ha els botons CA / ES / EN per canviar l'idioma en qualsevol moment.

🏠 Pàgina Inici — Arrencar els serveis

Aquesta és la pantalla principal del dashboard. Aquí pots:

  • Veure l'estat — un indicador de color mostra si el stack és En marxa (verd), Aturat (vermell) o Comprovant (groc).
  • Arrencar el stack — premeu el botó Arrencar. Els serveis Docker s'inicien en segon pla. Pot trigar 20-60 segons la primera vegada que s'arrenca.
  • Parar el stack — el botó Parar atura tots els contenidors Docker. Útil per estalviar recursos quan no es fa servir.
  • Accedir a un servei — els sis botons de la graella obren directament el servei al navegador (URLs de localhost). Estan desactivats si el stack és aturat.
⚠️
Els serveis s'accedeixen via localhost des del propi ordinador. Per accedir des d'altres dispositius (mòbil, televisió, portàtil), cal fer servir la IP de la xarxa local o la IP de Tailscale. Consulta la secció Xarxa.

🔒 Pàgina VPN

Des d'aquí pots gestionar la VPN en qualsevol moment, sense haver de reinstal·lar.

Quins proveïdors funcionen?

Gecko utilitza Gluetun com a contenidor VPN. No tots els proveïdors VPN funcionen amb aquesta configuració — cal que ofereixin WireGuard amb clau privada i adreça IP assignada. Els dos que funcionen de forma senzilla:

  • Mullvad — el més senzill. Genera una clau directament al web sense descarregar fitxers. ~5€/mes sense subscripció.
  • ProtonVPN — genera un fitxer de configuració WireGuard del qual s'extreuen les dades. Té pla gratuït.
⚠️
NordVPN, Surfshark i similars no funcionen directament amb aquest mètode. Gluetun els suporta però requereix variables d'entorn específiques per a cada proveïdor, fora de l'abast de l'assistent.

Mullvad — com obtenir les dades

1

Inicia sessió a my.mullvad.net

Ves a my.mullvad.net i accedeix al teu compte.

2

WireGuard configuration → Add new device

A la secció WireGuard configuration, escriu un nom (ex: gecko) i fes clic a Generate key.

3

Copia la clau privada i l'adreça

Mullvad mostrarà la clau privada (string llarg) i l'adreça IP assignada (10.68.x.x/32). Copia ambdues a Gecko i prem Activar VPN.

ProtonVPN — com obtenir les dades

1

Ves a account.proton.me → VPN → Downloads

Inicia sessió a account.proton.me i navega a VPN → Downloads → WireGuard configuration.

2

Genera i descarrega el fitxer .conf

Tria un país/servidor i descarrega el fitxer de configuració .conf.

3

Extreu PrivateKey i Address del fitxer

Obre el .conf amb el Bloc de notes. A la secció [Interface] trobaràs les dues dades que necessites:

[Interface]
PrivateKey = ABC123...xyz ← clau privada
Address = 10.2.x.x/32 ← adreça IP assignada

Copia ambdós valors a Gecko i prem Activar VPN.

Desactivar la VPN

Prem el botó Desactivar VPN. Gecko elimina les claus i passa al compose sense VPN automàticament.

IP de xarxa local

🚫
La IP local NO funciona fora de casa. Si intentes accedir a http://192.168.1.42:8096 des del mòbil amb dades mòbils, o des d'una altra xarxa, no funcionarà. Per accedir des de qualsevol lloc cal configurar Tailscale. Com configurar Tailscale →

🔌 Configurar Prowlarr — l'únic pas manual

Un cop acabat l'assistent de Gecko, l'únic que has de fer manualment és afegir indexadors a Prowlarr. Un indexador és un cercador de contingut (com Google però per a pel·lícules i sèries). Sense cap indexador, Radarr i Sonarr no trobaran res.

ℹ️
Prowlarr ja està connectat a Radarr, Sonarr i Lidarr. Gecko ho ha configurat automàticament. Quan afegeixes un indexador a Prowlarr, tots els serveis el veuran al moment — no cal fer res més.

Per afegir indexadors a Prowlarr:

1

Obre Prowlarr

Ves a localhost:9696 o fes clic al botó Prowlarr des del dashboard de Gecko.

2

Indexers → Add Indexer

Busca i afegeix els indexadors que vulguis. Prowlarr té centenars de font públiques i privades disponibles.

✓

Ja estàs llest

Els indexadors s'apliquen automàticament a Radarr, Sonarr i Lidarr. Ara pots demanar contingut des de Jellyseerr o afegir pel·lícules i sèries directament des de Radarr/Sonarr.

Com trobar bons indexadors

  • Busca "best public Prowlarr indexers [any actual]" a Google — hi ha llistes actualitzades regularment.
  • Pregunta a una IA (ChatGPT, Gemini…): "Quins son els millors indexadors públics per Prowlarr per a pel·lícules i sèries en català/espanyol?"
  • Consulta la documentació oficial de Prowlarr per veure el catàleg complet.
⚠️
Gecko no inclou ni recomana cap indexador concret. L'usuari és l'únic responsable del contingut al qual accedeix i de complir la legislació vigent al seu país.

📶 Accés local vs. accés extern

Hi ha dues maneres d'accedir als teus serveis, depenent d'on siguis:

Situació URL a fer servir Funciona? Necessita Tailscale?
A casa, al mateix Wi-Fi http://192.168.1.X:PORT ✓ Sí No
A casa, des del propi PC http://localhost:PORT ✓ Sí No
Fora de casa, mòbil amb dades http://192.168.1.X:PORT ✗ No Sí, obligatori
Fora de casa, altra xarxa Wi-Fi http://192.168.1.X:PORT ✗ No Sí, obligatori
Amb Tailscale, des de qualsevol lloc http://100.X.X.X:PORT ✓ Sí Sí (ja configurat)

🔗 Configurar Tailscale (accés extern)

Tailscale crea una xarxa privada virtual entre tots els teus dispositius. Un cop configurat, pots accedir als serveis de Gecko des de qualsevol lloc del món, com si estiguessis a casa. No cal obrir ports al router ni tenir IP pública.

✅
Tailscale és gratuït per a ús personal (fins a 3 usuaris i 100 dispositius). No hi ha límit de velocitat ni de dades.
1

Instal·la Tailscale a l'ordinador amb Gecko

Descarrega Tailscale per Windows, instal·la'l i inicia sessió amb Google, Microsoft o correu electrònic. Tailscale s'executa en segon pla.

2

Anota la IP de Tailscale del teu ordinador

A la barra de tasques, clica l'icona de Tailscale i veuràs la teva IP de Tailscale — és del format 100.x.x.x. Anota-la.

3

Instal·la Tailscale als altres dispositius

Descarrega Tailscale al mòbil (iOS / Android), portàtil o tablet. Inicia sessió amb el mateix compte. Activa la connexió.

4

Accedeix als serveis amb la IP de Tailscale

Des de qualsevol dispositiu amb Tailscale actiu, fes servir la IP 100.x.x.x en lloc de la IP local.

Com queden les URLs amb Tailscale

Si la teva IP de Tailscale és 100.64.1.25, les URLs serien:

# Jellyfin (reproductor)
http://100.64.1.25:8096

# Jellyseerr (peticions)
http://100.64.1.25:5055

# Radarr · Sonarr · Prowlarr · qBittorrent
http://100.64.1.25:7878 ← Radarr
http://100.64.1.25:8989 ← Sonarr
http://100.64.1.25:9696 ← Prowlarr
http://100.64.1.25:8090 ← qBittorrent
💡
Consell: Guarda les URLs de Tailscale als marcadors del navegador del teu mòbil per accedir-hi ràpidament. Les URLs de Tailscale sempre funcionen, tant si ets a casa com fora.

Tailscale i la VPN de Gecko

Tailscale i la VPN de les descàrregues de Gecko (ProtonVPN/Mullvad via Gluetun) no interfereixen entre elles. Tailscale és per accedir remotament als serveis. La VPN de Gecko és perquè les descàrregues de qBittorrent surtin xifrades. Pots tenir les dues actives simultàniament sense cap problema.
